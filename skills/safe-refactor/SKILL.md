# Safe Refactor

## Цель

Улучшить структуру при **нулевом** изменении поведения.

## Правила

1. Сначала зафиксируй baseline: тесты / ручной сценарий / скриншоты API response.
2. Один тип изменения за проход (rename **или** extract **или** move).
3. Не смешивать refactor с feat/fix в одном коммите, если пользователь не просил.
4. Публичные API / контракты трогать только с явного согласия.
5. После каждого шага — прогон релевантных тестов/typecheck.

## Чеклист перед финалом

- [ ] Поведение то же (тесты / smoke)
- [ ] Импорты и экспорты согласованы
- [ ] Нет мёртвого кода от старого пути
- [ ] Diff минимален относительно цели

## Запреты

- Не «улучшать» алгоритм, если задача — структура.
- Не переименовывать публичные символы без миграции.
- Не добавлять новые зависимости ради косметики.
